## 后端API接口

### 1. 游历板块接口
- **GET /api/travel/guides** - 获取所有导览内容
- **GET /api/travel/guides/:id** - 获取导览详情
- **GET /api/travel/panoramas** - 获取全景展示
- **GET /api/travel/routes** - 获取路线规划
- **GET /api/travel/search?keyword=xxx** - 搜索导览内容

### 2. 商城板块接口
- **GET /api/shop/hot** - 获取热门商品
- **GET /api/shop/series** - 获取系列商品
- **GET /api/shop/products** - 获取商品列表（支持分页）
- **GET /api/shop/products/:id** - 获取商品详情
- **GET /api/shop/categories/:category** - 按分类获取商品
- **GET /api/shop/search?keyword=xxx** - 搜索商品

### 3. 志愿者板块接口
- **GET /api/volunteer/activities** - 获取所有志愿活动
- **GET /api/volunteer/activities/:id** - 获取活动详情
- **GET /api/volunteer/activities/status/:status** - 按状态筛选活动
- **GET /api/volunteer/featured** - 获取推荐活动
- **POST /api/volunteer/activities/:id/join** - 参与活动
- **GET /api/volunteer/search?keyword=xxx** - 搜索活动

## 前端调用示例

在Vue组件中使用API：

```javascript
import { travelAPI, shopAPI, volunteerAPI } from '@/api';

// 获取热门商品
const fetchHotProducts = async () => {
  try {
    const response = await shopAPI.getHotProducts();
    if (response.success) {
      console.log('热门商品:', response.data);
    }
  } catch (error) {
    console.error('获取失败:', error);
  }
};

// 获取志愿活动
const fetchActivities = async () => {
  try {
    const response = await volunteerAPI.getActivities({ status: '进行中' });
    if (response.success) {
      console.log('志愿活动:', response.data);
    }
  } catch (error) {
    console.error('获取失败:', error);
  }
};
```

## 数据库设计

### 商品模型 (Product)
包含商品的所有属性，如名称、价格、库存、销量、评分等

### 志愿活动模型 (VolunteerActivity)
管理志愿活动的完整信息，包括时间、状态、参与人数等

### 导览模型 (Travel)
存储导览相关的路线、景点、难度等级等信息

## 部署说明

1. **启动MongoDB服务**
   ```bash
   mongod
   ```

2. **启动后端服务**
   ```bash
   cd backend
   npm install
   node seed.js  # 初始化数据
   npm run dev   # 启动开发服务器
   ```

3. **启动前端服务**
   ```bash
   npm run serve
   ```

## 注意事项

- 确保MongoDB服务正常运行
- 后端默认端口：3000
- 前端默认端口：8080
- API基础URL在前端 `src/api/index.js` 中配置
- 可根据需要修改 `.env` 文件中的配置参数
